# ゲームランチャーアプリ 要件定義書

## 1. 概要

本ドキュメントは、新規に開発するデスクトップ向け「ゲームランチャーアプリ」の要件を定義する。

## 2. 背景と目的

### 2.1. 背景

- PCゲーマーはSteam, Epic Gamesなど複数のプラットフォームでゲームを所有していることが多い。
- 各プラットフォームのクライアントを都度開いてゲームを起動するのは手間がかかる。
- OSのショートカット機能では、見た目が画一的でゲームを所有・プレイする楽しみが薄れる。

### 2.2. 目的

- PCにインストールされているゲームを、プラットフォームの垣根を越えて一元管理する。
- PS5のホーム画面のような、視覚的に豊かで魅力的なインターフェースを提供する。
- ゲームを選ぶ楽しさと、快適な起動体験を両立させる。
- 将来的にはSteamなどのストアで配布可能な品質のアプリケーションを目指す。

## 3. ターゲットユーザー

- 複数のPCゲームプラットフォームを利用しているゲーマー。
- デスクトップの見た目や、アプリケーションのUI/UXにこだわりを持つユーザー。

## 4. 機能要件

### 4.1. ゲーム管理機能

- **Steam連携**: Steamでインストール済みのゲームを自動的に検出し、ライブラリに表示する。
- **手動追加**: Steam以外のゲーム（Epic Games, GOG, スタンドアロン等）の実行ファイルを、ユーザーが手動で登録できる。
  - 登録時には、ゲームタイトル、実行ファイルパス、カバーアート用画像をユーザーが指定できる。
- **ゲーム情報表示**: ゲームのカバーアートをメインに、タイトルを表示する。
- **背景の動的変更**: 選択したゲームのプロモーションビデオ（入手可能な場合）やアートワークを背景に表示し、没入感を高める。

### 4.2. ゲーム起動機能

- ライブラリ上のゲームをクリックすることで、該当のゲームを起動できる。
  - SteamゲームはSteam経由で起動する。
  - 手動追加したゲームは、登録された実行ファイルを直接起動する。

### 4.3. UI/UX要件

- **ライブラリ画面**:
  - ゲームのカバーアートをグリッド状に表示する。
  - マウス操作で直感的に操作できる。
  - 初回起動時に、アプリケーションの主要機能やSteam APIキー設定への誘導など、ユーザーを支援するガイド機能があること。
- **デザイン**:
  - 全体的にモダンで洗練されたデザイン。
  - ゲームの世界観を邪魔しない、ダークテーマを基調とする。

## 5. 非機能要件

### 5.1. 性能

- アプリケーションの起動が高速であること。
- UIの操作がスムーズで、遅延を感じさせないこと（Jotaiなどの最適化された状態管理ライブラリの採用により実現）。

### 5.2. プラットフォーム

- Windows 10/11で動作すること。

### 5.3. セキュリティ

- ユーザーのPC環境に悪影響を与えないこと。
- 外部API（Steam Web API）との通信は安全に行うこと。

### 5.4. 法的要件・コンプライアンス

- Steam Web APIの利用規約を遵守すること。
- 各ゲームの著作権（カバーアート等）や商標を尊重し、アプリの機能は私的利用の範囲に留めることを明確にする。
- アプリ自体の名称やロゴは、既存のサービスと抵触しないオリジナルなものとすること。
